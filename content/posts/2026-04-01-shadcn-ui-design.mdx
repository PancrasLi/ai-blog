## 什么是 Shadcn/UI？

Shadcn/UI 是一个现代化的组件库，它不同于传统的包，而是提供了一套可复制到项目中的高质量组件代码。

### 核心特点

1. **可复制的组件** - 而非传统的 npm 包
2. **完全可定制** - 组件代码归你所有
3. **Tailwind CSS 驱动** - 利用 Tailwind 的强大能力
4. **Radix UI 基础** - 无障碍访问标准

## 设计思想

### 1. 组件优先

所有 UI 都由可复用的组件构成：
- Button（按钮）
- Card（卡片）
- Badge（标签）
- Separator（分隔符）

### 2. 变体系统

每个组件都有多种变体：

```typescript
// Button 变体
<Button variant="default" />    // 默认按钮
<Button variant="outline" />    // 描边按钮
<Button variant="ghost" />      // 幽灵按钮
<Button variant="link" />       // 链接按钮

// Badge 变体
<Badge variant="default" />     // 默认标签
<Badge variant="secondary" />   // 次要标签
<Badge variant="outline" />     // 描边标签
```

### 3. 响应式设计

使用 Tailwind 的响应式前缀：

```tsx
<div className="grid grid-cols-1 md:grid-cols-3 gap-6">
  {/* 移动端 1 列，中等屏幕以上 3 列 */}
</div>
```

## 性能优化

### 1. 文件大小
- 最小化 CSS（仅包含使用的类）
- 树摇优化组件导入
- 输出文件 ~50KB

### 2. 加载速度
- 静态生成所有页面
- 图片自动优化
- CSS 内联关键样式

### 3. 运行时性能
- React 18 Concurrent Features
- 自动代码分割
- 懒加载重组件

## 最佳实践

### ✅ 推荐做法

1. **使用语义化 HTML**
   ```tsx
   <Button asChild>
     <a href="/link">链接按钮</a>
   </Button>
   ```

2. **遵循颜色规范**
   ```tsx
   <div className="bg-background text-foreground">
     使用主题颜色
   </div>
   ```

3. **响应式优先**
   ```tsx
   className="text-sm md:text-base lg:text-lg"
   ```

### ❌ 避免的做法

1. 直接使用颜色值而不是变量
2. 混合多个设计系统
3. 过度定制组件（应该在项目级别调整）
