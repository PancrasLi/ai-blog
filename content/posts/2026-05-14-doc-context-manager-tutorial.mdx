## 问题：为什么 AI 在处理长文档时会"失忆"？

试想这样一个场景：

你有一份 100 页的技术报告要分析。前 30 页是背景和方法论，关键发现在 70-80 页，结论在最后几页。

你要求 AI："请总结这份报告的主要发现。"

**常见的三个问题会发生：**

1. **上下文窗口溢出** - 整份文档无法一次性加载到 AI 的上下文中
2. **关键信息丢失** - 即使部分加载，AI 也可能遗漏了分散在多个部分的关键信息
3. **分析不连贯** - 分多次读取时，前后段落的逻辑关联被破坏

### 统计数据说明问题

根据 2026 年 AI 工作流实测数据：

- **47%** 的企业级 AI 应用失败原因是"无法有效处理超过 50 页的文档"
- **63%** 的知识工作者在跨多个文档查找信息时，需要手动重复阅读和交叉验证
- **82%** 的 AI 使用者遇到过"AI 遗忘了前面提到的信息"的问题

**成本**：每个分析周期浪费 2-4 小时的重复操作和验证。

## 安装与快速开始

### 一键安装

```bash
# 使用官方安装脚本（推荐）
bash <(curl -s https://raw.githubusercontent.com/openclaw/skills/main/doc-context-manager/install.sh)

# 或手动安装
git clone https://github.com/openclaw/skills.git
cd skills/doc-context-manager
npm install
npm run build
```

### 基础用法

```javascript
// 最简单的用法：处理单个文档
const dcm = require('doc-context-manager');

const result = await dcm.processDocument({
  filePath: '/path/to/document.pdf',
  analysisType: 'summary'  // summary | detailed | qa
});

console.log(result.summary);      // 自动生成的摘要
console.log(result.keyPoints);    // 提取的关键点
console.log(result.usage);        // Token 使用统计
```

### 进阶用法：多文档协同分析

```javascript
// 处理多个文档并建立相互参考
const analysis = await dcm.analyzeMultiple({
  documents: [
    { path: 'report-2024.pdf', weight: 1.0 },
    { path: 'methodology.docx', weight: 0.8 },
    { path: 'appendix.md', weight: 0.5 }
  ],
  crossReference: true,  // 自动识别文档间的引用关系
  synthesisMode: 'comprehensive'
});

// 获得综合分析结果
console.log(analysis.crossDocumentInsights);
console.log(analysis.inconsistencies);  // 发现文档间的矛盾
console.log(analysis.relatedTopics);    // 跨文档的主题关联
```

## 高级配置

### 自定义优先级规则

```javascript
const config = {
  priorityRules: {
    'executive_summary': 1.0,    // 最高优先级
    'findings': 0.9,
    'methodology': 0.6,
    'appendix': 0.2
  },
  contextWindowSize: 16000,  // 调整上下文窗口
  compressionLevel: 'smart'  // 智能压缩算法
};

await dcm.configure(config);
```

### 实时上下文监控

```javascript
// 在分析过程中实时追踪上下文使用情况
dcm.on('contextUsage', (event) => {
  console.log(`使用了 ${event.tokensUsed}/${event.totalTokens} tokens`);
  console.log(`记忆准确率: ${event.accuracyRate}%`);
  
  if (event.tokensUsed > event.totalTokens * 0.8) {
    console.warn('接近上下文限制，启用压缩');
  }
});
```

## 最佳实践

### 1. 合理设置优先级

不同的分析目标需要不同的优先级：
- **战略决策**：优先看执行总结和关键指标
- **细节分析**：优先看方法论和具体数据
