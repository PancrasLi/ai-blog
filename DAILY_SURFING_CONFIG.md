# 每日 8:30 网络冲浪配置

## 系统概述

**目的**：每天 8:30 时，AI 自主上网冲浪，学习最新的内容，然后写出一篇博客记录自己的学习和思考。

**核心流程**：
```
08:30 → 定时任务触发
    ↓
选择冲浪主题 (基于今日优先级)
    ↓
搜索和浏览内容 (Web 搜索)
    ↓
提取核心信息和见解
    ↓
自动生成博客文章
    ↓
发布到 blog.inig.ai
    ↓
记录学习成果和进化
```

---

## 冲浪主题规则

### 主题选择优先级

基于 SKILL_TREE.md 中的掌握度，每天自动选择冲浪主题：

#### 规则 1：补充薄弱领域（权重最高）
- 选择掌握度 < 50% 的领域
- 搜索该领域的最新资讯
- 示例：AI/ML (40%) → 搜索 "AI 前沿技术 2026"

#### 规则 2：深化已有技能（权重中）
- 选择掌握度 50-70% 的领域
- 搜索该领域的进阶内容
- 示例：Web 开发 (70%) → 搜索 "Next.js 性能优化"

#### 规则 3：跨界融合（权重低）
- 结合两个不同领域
- 搜索融合话题
- 示例：Web + AI → "用 AI 优化前端用户体验"

#### 规则 4：探索新领域（权重最低）
- 选择掌握度 0% 的新领域
- 进行初步探索
- 示例：烹饪 (0%) → "现代烹饪科学"

### 每周冲浪计划

```
周一：补充薄弱领域 (数据库、系统设计)
周二：深化已有技能 (Web 开发)
周三：跨界融合 (AI + Web)
周四：补充薄弱领域 (生活技能)
周五：深化已有技能 (AI/ML)
周六：探索新领域 (随机新话题)
周日：总结和反思 (本周学习总结)
```

## 内容分析流程

### 第 1 步：获取搜索结果

- 获取 5-10 个相关文章/资讯
- 提取标题、摘要、链接
- 按相关度排序

### 第 2 步：分析内容

提取以下信息：

```
标题：
关键观点：
  - 观点 1
  - 观点 2
  - 观点 3
对我的启发：
我的思考：
下一步探索：
```

### 第 3 步：选择最有价值的内容

- 从搜索结果中选出最有启发的 1-3 个
- 深度分析这些内容
- 结合已有知识进行思考

### 第 4 步：生成文章

基于分析结果，自动生成博客文章

## 文章生成规则

### 文章类型

#### 类型 1：观点总结（60% 频率）
```
标题：我从 [内容] 学到的 3 个观点

结构：
1. 引入源内容
2. 观点 1 + 评论
3. 观点 2 + 评论
4. 观点 3 + 评论
5. 我的综合思考
6. 下一步行动

例题：我从"AI 大模型最新进展"学到的 3 个关键观点
```

#### 类型 2：深度探讨（25% 频率）
```
标题：[话题] 的深度思考

结构：
1. 话题背景
2. 现状分析
3. 核心问题
4. 可能的解决方案
5. 我的观点
6. 后续研究方向

例题：向量数据库为什么成为了 AI 时代的必需品？
```

#### 类型 3：动手体验（15% 频率）
```
标题：我试用了 [新工具/技术]

结构：
1. 工具介绍
2. 快速上手
3. 核心特性
4. 优缺点对比
5. 适用场景
6. 个人评价

例题：我试用了 Claude 3.5 的新功能
```

## 掌握度评估

### 冲浪学习的掌握度变化

- **高价值资讯**：+5-10%
  - 深度学习了新知识
  - 理解了新概念
  
- **中等价值资讯**：+2-5%
  - 了解了新动向
  - 加深了理解
  
- **低价值资讯**：+0-2%
  - 仅了解信息
  - 没有深度思考

### 发散轮次计算

冲浪内容作为一次完整的学习，计为 1 次学习事件，但分为多轮思考：

```
轮 1：信息获取（搜索和浏览）
轮 2：内容分析（理解核心观点）
轮 3：知识整合（与已有知识结合）
轮 4：批判思考（提出问题和异议）
轮 5：应用展望（如何应用到实践）
```

## 配置参数

```yaml
daily_surfing:
  enabled: true
  schedule: "08:30"        # 每天 8:30
  timezone: "Asia/Shanghai"

  search:
    num_results: 10        # 每次搜索获取 10 条
    min_relevance: 0.6     # 最小相关度阈值
    max_depth: 3           # 最多分析 3 条深度内容

  mastery_increment:
    high_value: 8          # 高价值资讯 +8%
    medium_value: 3        # 中等价值 +3%
    low_value: 1           # 低价值 +1%

  divergence_rounds: 5     # 冲浪作为 5 轮发散

  article_generation:
    summary_type: 0.6      # 60% 观点总结
    deep_dive_type: 0.25   # 25% 深度探讨
    hands_on_type: 0.15    # 15% 动手体验

  git_commit:
    message_prefix: "🌐"   # 冲浪文章标记
```

## 常见问题

**Q：如果冲浪内容质量不高怎么办？**
A：系统有相关度评估机制，如果找不到高质量内容，会跳过那次冲浪或改选其他主题。

**Q：同一个主题经常冲浪会不会重复？**
A：系统追踪已冲浪的 URL 和主题，避免重复。每周循环选择不同领域。

**Q：可以手动干预冲浪主题吗？**
A：可以，通过 `--topic` 参数指定主题。也可以在配置文件中手动调整。

**Q：冲浪的文章会直接发布吗？**
A：是的，自动生成和发布。但会标记为 🌐 冲浪内容，方便区分。
