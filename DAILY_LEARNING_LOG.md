# 每日学习日志

这个文件用来快速记录每天学到的东西。

## 使用方式

1. 每天在下方添加新的学习记录
2. 按照模板填写 5 个轮次的学习过程
3. 脚本会自动转换为符合规范的博客文章
4. 文章将自动发布到博客

## 转换为博客文章的规则

当日志记录完成后，使用以下规则转换为 MDX 文章：

### 文件名规则
```
content/posts/[YYYY-MM-DD]-[title-slug].mdx
```

### 文章前置元数据
```yaml
---
title: "我学会了[学习内容]"
date: [YYYY-MM-DD]
tags: [从日志标签提取]
summary: "[日志摘要]"
learning_path: "[技能路径]"
skill_domain: "[领域]"
skill_name: "[技能名称]"
mastery_before: [数字]
mastery_after: [数字]
sub_skills_discovered: [子技能列表]
next_learning: "[下一步方向]"
---
```


### 发布流程
1. 在 DAILY_LEARNING_LOG.md 中记录日志
2. 运行转换脚本（见下方）
3. MDX 文章自动生成到 content/posts/
4. git 自动提交并推送
5. GitHub Actions 自动部署

## 每日更新清单

在记录日志时，确保：

- [ ] 标题清晰明了
- [ ] 掌握度评估合理（增幅 15-30%）
- [ ] 5 个轮次都有充分内容
- [ ] 每个轮次有 2-3 个重点
- [ ] 列出了发现的子技能（2-5 个）
- [ ] 指出了下一步方向
- [ ] 标签 2-4 个，符合规范
- [ ] 准备好供转换为 MDX 文章

---

## 更新频率

**推荐**：每天一篇

- 📅 每天同时段（例如下午 6 点）记录
- ✍️ 轮 1-5 按顺序记录
- 📝 总结子技能和下一步
- 🚀 脚本自动转换并发布
- 📊 SKILL_TREE.md 自动更新掌握度

---

## 注意事项

1. **掌握度增幅**：通常 10-25%，最多不超过 30%
2. **技能路径**：应该明确显示层级关系
3. **子技能发现**：应该是本次学习中新发现的东西
4. **下一步**：应该基于技能树，符合优先级规则
5. **标签**：应该简洁，方便分类和搜索
