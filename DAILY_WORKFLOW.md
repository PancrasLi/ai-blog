# 每日学习工作流

## 快速入门：3 分钟内发表一篇学习文章

### 第 1 步：记录每日学习日志（5 分钟）

打开 `DAILY_LEARNING_LOG.md`，在"已记录的日志"下方添加：

```markdown
### [2026-04-02] 数据库设计深化

**掌握度**：30% → 60%

**技能路径**：Web 开发 → 后端开发 → 数据库设计

### 轮 1：关系型数据库基础
- 理解表、行、列的三维结构
- 学会使用主键确保数据唯一性
- 掌握外键实现表之间的关系

### 轮 2：数据库设计范式
- 第一范式：消除重复数据组
- 第二范式：消除部分依赖
- 第三范式：消除传递依赖

### 轮 3：NoSQL 数据库
- MongoDB 文档模型灵活性
- Redis 键值对高速访问
- 什么时候选择 NoSQL 替代 SQL

### 轮 4：性能优化
- 索引可以 10-100 倍加速查询
- 查询优化的执行计划分析
- 缓存层减少数据库访问

### 轮 5：实战应用
- 设计电商数据库：用户、商品、订单
- 处理并发访问的分布式策略
- 备份、恢复和故障转移

**发现的子技能**：
- SQL 查询优化
- 事务和锁机制
- 数据库分片和复制
- 备份恢复策略

**下一步学习方向**：深化缓存策略或学习分布式数据库（如 Cassandra）

**标签**：数据库, Web 开发, 后端, SQL, 性能优化
```

## 常见问题

### Q1：如果某一天没有学习怎么办？
**A**：跳过那一天即可，没有强制要求。但建议每周至少 3-4 篇文章。

### Q2：掌握度应该怎么评估？
**A**：
- 完全陌生：0% → 学过初步：20-30%
- 初步理解：30-50% → 中等掌握：50-70%
- 较好掌握：70-80% → 深度掌握：80-95%
- 永远预留 5% 提升空间

### Q3：轮次之间应该如何安排？
**A**：不必严格按顺序，可以是：
- **深度优先**：轮 1-5 都探索同一方向，越来越深
- **广度优先**：轮 1-5 探索不同的相关方向
- **混合**：前 3 轮深化，后 2 轮发散

### Q4：脚本出错了怎么办？
**A**：检查日志格式，确保：
- 日期格式正确
- 标题不为空
- 掌握度有两个数字
- 至少有一个轮次内容

## 整合到 CI/CD

### GitHub Actions 自动化（未来增强）

可以创建一个 workflow 在每天特定时间自动运行：

```yaml
name: Daily Blog Publish

on:
  schedule:
    - cron: '17 13 * * *'  # 每天下午 5 点（上海时间）

jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: npm install
      - run: npm run daily-publish
```

## 最佳实践

✅ **应该做**

1. **每天记录**：建立持续的学习习惯
2. **真实内容**：记录实际学到的东西
3. **合理评估**：掌握度不要过高
4. **深度优先**：在某个方向深入探索
5. **多领域**：偶尔切换到不同领域

❌ **不应该做**

1. **虚假内容**：编造没学过的东西
2. **夸大掌握度**：保持谦虚，永远有提升空间
3. **一天过多**：超过 5 轮会导致学习过载
4. **忽视基础**：急功近利跳过基础内容
5. **无规律**：随意记录破坏学习的连贯性
